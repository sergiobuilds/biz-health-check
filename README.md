# biz-health-check

사업자등록번호 하나로 "이 사업자, 실제 문제 없나"를 확인하는 실사 사실 조회 스킬.
k-skill(NomaDamas/k-skill) 기여용으로 개발 중 — 스킬 본체는 `SKILL.md`, 구현은 `scripts/`.

## 핵심 원칙

점수·등급·해석 라벨 없음. 조회된 사실 + 출처 + 조회시각만 병렬. 실패는 크래시 대신 `unavailable`/`needs-key` 강등.

## 빠른 실행

```bash
DATA_GO_KR_KEY=<공공데이터포털 키> python3 scripts/biz_health_check.py 124-81-00998 --name 삼성전자
python3 -m pytest tests/ -q   # 오프라인 단위 테스트 (네트워크 차단)
```

## k-skill 기여 메모

- 1차 PR은 BYO 키(`DATA_GO_KR_KEY`)로만. proxy route(연금·금융위)는 운영자 협의 후 후속 분리.
- 합본 vs 단품 분리, 네이밍은 운영자 회신 반영 예정.

## v2 로드맵 — 소규모 사업장(인허가 업종) 확장

목표: 법인 중심 5종에 더해, 동네 사업장(식당·숙박 등 인허가 업종)도 "영업 중/휴업/폐업, 업력, 주인 교체"가 한 방에 나오게 한다.

- 백본 후보 1 — **행정안전부 개방표준(표준데이터셋)**: 전국 일반음식점(data.go.kr 15096283), 전국 숙박업소(15155657). `api.data.go.kr/openapi/tn_pubr_public_*_api` 패턴, data.go.kr 키 + 활용신청(자동승인). **엔드포인트 slug가 활용신청 승인 페이지에서만 공개**되는 구조 확인(2026-06-11) — 신청 후 배선.
- 백본 후보 2 — **LOCALDATA(지방행정인허가, localdata.go.kr)**: 단일 키로 ~190 인허가 업종 + 변동분 동기화. 별도 회원가입·키 발급 필요, 벌크 동기화 지향(상호 단건 검색 비적합). 2026-06-11 현재 사이트 접속 불가(타임아웃) — 복구 후 재평가.
- 매칭 한계(정직): 인허가 데이터에는 사업자등록번호가 수록되지 않아 상호+소재지 문자열 매칭만 가능 — 체납 명단과 같은 identity_note 방식으로 동일성 단정 금지.
- 모니터링(watchlist+diff+알림)은 단발 조회와 성격이 달라 별도 스킬/인프라로 분리한다.
